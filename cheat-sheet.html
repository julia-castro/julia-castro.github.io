Git Cheatsheet

Making Changes

git status

Lists new or modified files to committed.

git add [file]

Stages the file to be committed

git add .

Stages all files to be committed.

git reset [file]

Unstages the file but preserves its content.

git commit -m [commit messsage]

Saves snapshot of the file in the version history.

git commit -v

Saves snapshot of the file in the version history but allows you to record a longer commit message (v is for verbose).

git push origin master

Pushes changes from your local repository to GitHub.

Forking and Cloning

To fork a repository on GitHub:

  1. Go to the master repository you want to fork.
  2. Click on the fork button on the top right hand corner.
  3. GitHub will ask you “Where should we fork this repository?” Click on your username.

To clone a repository to your machine using Git:

  1. Copy the HTTPS clone URL.
  2. Open up your terminal.
  3. Navigate to the directory you want to save the repository.
  4. Enter ‘git clone ‘ and paste the URL i.e. git clone https://github.com/tekd/phase-0-unit-1.git

Setting Up Remotes

You'll only need to do set up a remote once! Make sure you're in the correct directory for fetching.

git remote -v

Check your current remote. This should link to your personal fork.

git remote add upstream

Set up a new remote

Fetching and Merging

$ git remote -v

origin refers to the name of the remote repo

master refers to the name of the branch you are pushing

Fetch the Changes

Fetching from a remote repository will bring those changes into your local repository but they are not merged yet.

$ git fetch upstream

# Grab the upstream remote's branches

$ git branch -va

# List all local and remote-tracking branches

Merge the changes

$ git branch

checks which branch you are currently in

git checkout master

switch to your master branch

git merge upstream/master

Merges upstream's master into your local forked repository. If you did not have any un-synced commits, git will perform a "fast-forward".If git will opens up a Sublime window and asks for a commit message, save and quit, or add a commit message and then save and quit.
