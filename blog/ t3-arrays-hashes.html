ARRAYS

Array: basically a list in your computer. It is a collection that stores a list of values in a single variable. Arrays can consist of anything: numbers, strings, other arrays, etc...

colors = ["pink", "red","blue"]
numbers = [5, 6, 7]
etc = [24, "new york", [true, false]]

All the examples above are arrays. Each slot in the list acts like a variable and points to a particular object.

Each slot also has an index number. The first element is index 0, not 1. So in the array colors in the example above, index 0 points to pink, index 1 points to red and index 2 points to blue. Easy enough to understand, if not to remember

Because the slots act as variables, you can assign to them as well.

countries = [ ]
countries[4] = "USA"
countries[1] = "Canada"
countries[2] = "Mexico"
countries[1] = "Italy"

puts countries

Italy
Mexico
USA

As you can see, you don't have to assign the slots in order, and you can even re-assign. If you leave a slot empty, it is filled with nil by default.

HASHES

Hashes are very similar to arrays in that they have slots that point to various objects. The biggest difference is that arrays are indexed with ordered numbers, and hashes can be indexed with ANY OBJECT! not just numbers!

LET'S COMPARE THE TWO:

Array:
colleges = ["UC Santa Cruz", "Fordham", "Kings College London"]

Hash:
colleges = {
"California" => "UC Santa Cruz",
"New York" => "Fordham",
"England" => "Kings College London"
}

So in the Array, each element can only be accessed via an ordered, numbered slot. But in the Hash, "UC Santa Cruz" can be assigned to ANY variable, not just a number. In other words, in Arrays, the keys in the key-value pairs are limited to integers. In Hashes, the keys in the key-value pairs are NOT limited to integers. On one hand, Hashes are more descriptive and specific because you can assign key-value pairs that are actually informative. But on the other hand, because Hashes aren't ordered data, it could be difficult to get data out the Hash in any ordered way.
