OO vs. Functional Programming

4.5.15

In my short time programming I have only been exposed to object-oriented programming in my attempts to learn Ruby. I've learned that everything is an object, and that these objects interact with eachother using methods. Objects also have attributes, which are just data.

For example, in

"Julia".length
# ==> 5

the "Julia" object is a string with a .length method and a length attribute of 4.

Object-oriented programming seems to be based on a simple concept, but in researching object-oriented programming vs. functional programming I've learned there is more to it.

As the term suggests, object-oriented programming emphasizes objects and so we define these objects in terms of data that they encapsulate and hide from the outside world and the behaviour that the objects expose to the outside world, which operates on that data.

Here's an example of a rectangle class. It has hidden data, width and height, and doesn't expose those anywhere to the outside world. But you can ask it things, like are you a square? or you if you want it to be a square, you can mutate it into a square.

class Rectangle
def initialize(width, height)
@width = width
@height = height
end

def square?
@width == @height
end

def square!
if @width > @height
@height = @width
else
@width = @height
end
end
end

Functional Programming emphasizes functions. Instead of data encapsulation,it has you define very simple public data structures.

Here we have a rectangle structure, which has width and height, which are publicly accessible everywhere. You then define functions which accept those data structures and then return data structures as a result. The square! method accepts a rectangle, but instead of modifying it, it returns a new rectangle.

Rectangle = Struct.new(:width, :height)

def square?(rectangle)
rectangle.width == rectangle.height
end

def square!(rectangle)
if rectangle.width > rectangle.height
Rectangle.new(rectangle.width, rectangle.width)
else
Rectangle.new(rectangle.height,rectangle.height)
end
end
