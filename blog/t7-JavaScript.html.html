Javascript Objects vs. Ruby Hashes

4.14.15

As I've been learning Javascript, I'm realizing that I'm having an easier time of it because I already have a foundation in Ruby. The two languages share a lot of the same concepts, but differ only in syntax (which admittedly, does mess me up quite a bit).

One very blatant similarity between the two languages are Javascript Objects and Ruby Hashes.

Javascript objects are data structures compised of different properties. They are declared as variables, and like Ruby hashes, are a way of organizing key/value pairs. The example below demonstrates how objects are created using the literal technique.

var me = {
name: "Julia",
age: 24
}

The object "me" has two properties. To access each property:

console.log(me.name)
console.log(me.age)

You can delete a property by writing delete me.name
To declare a new property, write me.gender = "female"

Ruby hashes look extremely similar. Values are assigned to keys using =>

me = { "name" => "Julia",
"age" => 24
}

You can access values in a hash just like an array.

puts me["name"]

You can delete a key-value pair by writing me.delete("name")
To add a new key-value pair write me["gender"] = "female"
